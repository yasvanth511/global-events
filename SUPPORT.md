# Support

Use GitHub Issues for reproducible bugs and focused feature requests.

Before opening an issue:

- Search existing issues.
- Use the appropriate issue form.
- Include reproduction steps and environment details for bugs.
- Remove credentials, private data, and personal information.

Security vulnerabilities must be reported privately according to
`SECURITY.md`, not through a public issue.

This project is maintained on a best-effort basis. There is no guaranteed
response time or service-level agreement.
