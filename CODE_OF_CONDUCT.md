# Code of Conduct

## Our Standard

Contributors and maintainers must communicate respectfully and constructively.
Healthy participation includes:

- Giving useful technical feedback without personal attacks.
- Respecting different experience levels and viewpoints.
- Keeping discussions relevant to the project.
- Accepting correction and taking responsibility for mistakes.
- Protecting private and security-sensitive information.

Harassment, threats, discrimination, deliberate disruption, doxxing, or
publishing another person's private information are not acceptable.

## Enforcement

Report conduct concerns privately to the repository owner through their GitHub
profile. Do not use a public issue when the report includes sensitive personal
information.

The maintainer may edit or remove comments, close issues, reject
contributions, or temporarily or permanently restrict participation when
behavior violates this policy.

Enforcement decisions should be proportionate, documented privately when
appropriate, and focused on protecting project participants.
